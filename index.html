The Java Collection Framework

java最初的发行版仅仅为最有用的数据结构（Vector，Stack, Hashtable, BitSet）提供了一小组类.JAVA SE 1.2才提供完备的数据结构

separating Collections interface and Implementation

java collection library可以分离为interface和implementation。

以queue举例来说

当你在程序中使用queue，collection构建时不用纠结于那种实现方式，在构建collection对象时再使用具体的类。

Use the interface type to hold the collection reference

The Collections Interface

collections class的基本接口就是collections。

Concrete Collections

Maps

map存储了key/value pairs,可以根据key找到value，key是唯一的。

Map interface 提供了三种collection view,允许map的内容展示为

  • set of keys
  • collections of values
  • set of key-value mapping

Map的排序为迭代器map里的iterator返回elelment的排序。

map的key值最好选immutable object，如String Double。

所有通过map实现的类最好提供两个标准的构造器

  • 一个无参构造器创建空的map
  • 另一个构造器有map的类型参数-创建了由相同的key/value作为参数的新的map。

map基本操作

map两种一般实现方式

  • hashMap对key值进行hash，无序。继承Map<K,V>, Cloneable, Serializable接口
  • treeMap使用Red-Black tree对key进行排序。继承NavigableMap<K,V>, Cloneable, java.io.Serializable

Interface vs Abstract

发表于 2018-12-15 | | 热度 ℃

Abstract

  • abstract class与interface相似，你不能实例化它们，而且它们可能会包含许多声明或是没有声明的混杂的方法。
  • 对于abstract class，可以声明不是static和final的field，而且能够定义public，private，protected 的具体方法。
  • 一个abstract class 仅可以extend一个父类，但是它可以implement多个interface
  • 如果抽象类被继承了，子类依旧能够提供对父类抽象方法的实现，如果不能的话，子类必须声明为抽象类。
