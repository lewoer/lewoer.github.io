Composition

发表于 2018-07-20 | 分类于 java | | 热度 ℃

任何软件系统的设计，其中的一个基本的活动便是要建立各个类之间的联系。关联类之间的两个基本的方式就是 inheritance和 composition。
虽然编译器和Java虚拟机（JVM）在使用继承时会为你做很多工作，但在使用组合时也可以获得继承功能。

Inheritance是 IS-A关系，而 composition是 HAS-A关系。

You do composition by having an instance of another class C as a field of your class, instead of extending C

Static vs Instance (Method,variable)

发表于 2018-07-17 | 分类于 java | | 热度 ℃

Static Methods

  • Static methods声明使用 static修饰符
  • Static methods不能直接访问 instance methods 和 instance variable
  • 通常用于一些操作，这些operations不需要从类的实例获取数据（from “this”）
  • 在 static methods我们不能使用 this关键字
  • 当你看见一个方法 不使用instance variable，这个方法应该被声明为 static method。比如说 main是一个static method。在应用启动的时候，被JVM调用。

Method Overriding vs Overloading Recap

发表于 2018-07-17 | 分类于 java | | 热度 ℃

Method Overloading

  • 方法重载意味着在一个类中提供两个或三个不同的方法，它们有相同的名字，但是却参数不同
  • 方法的返回类型可能相同，可能不同，允许我们 reuse相同的方法名字。
  • overloading非常方便，减少了复制代码而且部门不必去记住很多方法的名字。

  • overloading与多态(polymorphism)没有什么关系，但是开发者却认为是compile time polymorphism。

    编译器基于方法名字，返回类型和参数表决定调用哪种方法

  • 我们可以重载 static and instance方法。

This vs Super

发表于 2018-07-17 | 分类于 java | | 热度 ℃

先讨论 this and super 关键字

  • 关键字 super用来访问或调用父类成员(variable and member)
  • 关键字 this用来访问和调用当前类成员(variable and member).当我们有一个参数与实例变量(field)名字相同时，需要用到。

NOTE: 除了 static area(static block or static method),我们都能使用这两个关键字。否则会引发 compile-time errors
