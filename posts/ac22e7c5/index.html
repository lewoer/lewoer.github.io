web基础

1997年1月公布的HTTP/1.1是目前主流的HTTP协议版本, 最新标准是RFC2616

网络基础Tcp/Ip

Tcp/Ip协议族: 与互联网县相关联的协议集合(tcp,ip,http,ftp…)

Tcp/Ip分层管理

  • 应用层: 决定了向用户提供应用服务是通信的活动. FTP,DNS(域名系统),HTTP
  • 传输层: 提供属于网络连接中两台计算机之间数据传输. TCP,UDP
  • 网络层: 处理网络上流动的数据包(data packet).IP
  • 链路层: 处理连接网络的硬件部分.

tcp/ip通信传输流

这种将信息包装起来的方法为封装(encapsulate)

IP TCP DNS

负责传输的ip协议

IP协议(Internet protocol)位于网络层, 作用将数据包传递给对方.两个重要条件

  • IP地址: 节点被分配到的地址
  • MAC地址(Media Acess control Address): 网卡的固定地址

IP间的通信依赖MAC地址: 通常需要中转多次,才能连接到对方.中转时候,利用 ARP(地址解析协议),根据IP地址反查mac地址,查找下一目标

routing(路由选择): 在到达通信目标前的中转过程中，那些计算机和路由器等网络设备只能获悉很粗略的传输路线

确保可靠性的TCP协议

tcp位于传输层,提供可靠的字节流服务(Byte Stream service).将packet分割为较小的segment(报文段)进行传输, 为了传输可靠,采用三次握手(three-way hand-shaking)策略.

握手过程采用TCP标志(flag)

  • SYN(synchronize)
  • ACK(acknowledgement)

若在连接过程中中断连接,会以相同的顺序继续发相同数据包.

域名解析的DNS

DNS提供通过域名查找ip,或者通过ip查找域名

各种协议与Http协议的关系

URI 和 URL

URIs, URLs, and URNs: Clarifications and Recommendations 1.0

URI( Uniform Resource Identifier )统一资源标识符

  • Uniform: 规定统一的格式方便处理多种类型的资源,新增协议方案(ftp,http)
  • Resource: 可标识的任何东西

  • Identifier: 标识可标识的对象

    标准的URI协议方案有30多种,由 IANA管理颁布.

使用URI标识摸个互联网资源,URL标识资源的地点.URL是URI的子集.

URI格式

  • 绝对URI

  • 相对URI

/catalogue/phones?os=android#samsung

HTTP是不保存状态的协议

http为了更快的处理大量事务,设置为无状态协议(stateless).因此引入了cookie来实现保存状态的功能.

HTTP方法

w3c方法定义

  • GET: 获取资源
  • POST: 传输实体的主体(主要目的不是获取相应的内容)
  • PUT: 传输文件(无验证机制,存在安全问题)
  • HEAD: 获取报文首部
  • DELETE:按照请求URI删除指定的资源(无验证,不安全)
  • OPTIONS:询问支持的方法
  • CONNECT: 使用隧道协议建立连接

    与代理服务器建立通信的时候简历里隧道,实现用隧道协议(SSL+TLS)加密后传输.

持久连接,管线化(pipeling)

使用cookie的状态管理

Cookie技术通过在请求和响应报文中写入Cookie信息来控制客户端的状态.

编码提升传输效率

http在可以在传输中编码以提升传输效率

报文主体(message)和实体主体(entity)的差异

  • 报文

    HTTP通信的基本单位,8位组字节流组成,通过HTTP通信传输

  • 实体

    作为请求或响应的有效载荷数据（补充项被传输，其内容由实体首部和实体主体组成

通常，报文主体等于实体主体。只有当传输中进行编码操作时，实体主体的内容发生变化，才导致它和报文主体产生差异.

压缩传输的内容编码

内容编码: 应用在实体内容上的编码格式,保持实体信息原样压缩,编码后的实体由客户端接受并解码

常见的内容编码

  • gzip(GNU zip)
  • compress(UNIX标准压缩)
  • deflate
  • identity(不进行编码)

分块传输编码

分块传输编码: 传输大容量数据,将数据分割为多块,让浏览器逐步显示页面,这种实体主体分块的功能为分块传输编码

每一块用16进制标记块的大小,实体最后使用0标记.

发送多种数据的多部分对象集合

HTTP协议中采纳了多部分对象集合，发送的一份报文主体内可含有多类型实体。通常是在图片或文本文件等上传时使用.

  • multipart/form-data : 在 Web 表单文件上传时使用。
  • multipart/byteranges : 状态码 206（Partial Content，部分内容）响应报文包含了多个范 围的内容时使用。

使用时候,需要在首部字段加入Content-type

获取部分内容的范围请求(range request)

解决下载中断问题,执行范围请求,需要用到首部字段Range指明指定资源的byte范围

内容协商返回最合适的内容(Content Negotiation)

内容协商机制是指客户端和服务器端就响应的资源内容进行交涉，然 后提供给客户端最为适合的资源。内容协商会以响应资源的语言、字 符集、编码方式等作为判断的基准。

首部字段,判断基准

  • Accept
  • Accept-Charset
  • Accept-Encoding
  • Accept-Language
  • Content-Language

内容协商技术

  • 服务器驱动协商
  • 客户端驱动协商
  • 透明 协商(两者的接合体)

返回结果的HTTP状态码

状态码的职责是当客户端向服务器端发送请求时，描述返回的请求结果。借助状态码，用户可以知道服务器端是正常处理了请求，还是出现了错误.

下面介绍常用的14种状态码

2XX成功

  • 200 OK
    1. 表示从客户端发来的请求在服务器端被正常处理了。
    2. 在响应报文内，随状态码一起返回的信息会因方法的不同而发生改变
  • 204 No Content
    1. 该状态码代表服务器接收的请求已成功处理，但在返回的响应报文中 不含实体的主体部分。
    2. 不允许返回任何实体的主体。
    3. 一般在只需要从客户端往服务器发送信息，而对客户端不需要发送新信息内容的情况下使用。
  • 206 Partial Content

    该状态码表示客户端进行了范围请求，而服务器成功执行了这部分的 GET 请求

3XX重定向

  • 301 Moved Permanently

    永久性重定向.

  • 302 Found

    临时性重定向。

  • 303 see other

    该状态码表示由于请求对应的资源存在着另一个 URI，应使用 GET 方法定向获取请求的资源

  • 304 Not Modified

    该状态码表示客户端发送附带条件的请求 2 时，服务器端允许请求访 问资源，但未满足条件的情况

  • 307 Temporary Redirect

    307会遵照浏览器标准，不会从POST变成GET

4XX 客户端错误

  • 400 Bad Request

    该状态码表示请求报文中存在语法错误

  • 401 Unauthorized

    该状态码表示发送的请求需要有通过HTTP认证(BASIC 认证、 DIGEST 认证)的认证信息

  • 403 Forbidden

    该状态码表明对请求资源的访问被服务器拒绝了

  • 404 Not Found

    该状态码表明服务器上无法找到请求的资源。除此之外，也可以在服务器端拒绝请求且不想说明理由时使用。

5XX

  • 500 Internal Server Error

    该状态码表明服务器端在执行请求时发生了错误

  • 503 Service Unavailable

    该状态码表明服务器暂时处于超负载或正在进行停机维护，现在无法处理请求.

与HTTP协作的Web服务器

用单台虚拟主机实现多个域名

域名通过DNS服务映射到IP地址访问目标网站.如果一台服务器托管好多个网站,ip地址一样,因此需要搞清楚,访问那个域名.

在相同的 IP 地址下，由于虚拟主机可以寄存多个不同主机名和域名的Web网站，因此在发送HTTP请求时，必须在Host首部内完整指 定主机名或域名的 URI.

通信数据转发程序：代理、网关、隧道

代理

代理是一种有转发功能的应用程序，它扮演了位于服务器和客户端“中间人”的角色.接收客户端发送的请求后转发给其他服务器

在 HTTP通信过程中，可级联多台代理服务器。请求和响应的转发会 经过数台类似锁链一样连接起来的代理服务器

使用代理原因:

  • 利用缓存技术减少网络带宽的流量
  • 组织内部针对特定网站的访问控制

代理的基准分类:

  • 缓存代理(Caching Proxy)

    代理转发响应时，缓存代理(Caching Proxy)会预先将资源的副本(缓存)保存在代理服务器上。

  • 透明代理(Transparent Proxy)

    转发请求或响应时，不对报文做任何加工的代理类型被称为透明代理.对报文内容进行加工的代理被称为非透明代理

网关

利用网关可以由HTTP请求转化为其他协议通信,提供非HTTP协议服务。

利用网关能提高通信的安全性,在客户端与网关之间的通信线路上加密以确保连接的安全.网关连接数据库,使用sql语句查询.网站结算,网关与信用卡结算系统联动.

隧道

隧道的目的是确保客户端能与服务器进行安全的通信,隧道本身不会去解析HTTP请求.

保存资源的缓存

缓存是指代理服务器或客户端本地磁盘内保存的资源副本。利用缓存可减少对源服务器的访问，因此也就节省了通信流量和通信时间。

缓存的有效期

即使存在缓存，也会因为客户端的要求、缓存的有效期等因素，向源服务器确认资源的有效性。若判断缓存失效，缓存服务器将会再次从 源服务器上获取“新”资源。

缓存不仅可以存在于缓存服务器内，还可以存在客户端浏览器中
