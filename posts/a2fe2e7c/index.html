Basic Input & Output

发表于 2019-01-19 | | 热度 ℃

Byte stream

程序使用 8位字节执行输入和输出，所有的字节流类都继承了inputStream outputStream.

使用FileInputStream,FileOutputStream.

一定要关闭流stream

  1. 关闭不再使用的流很重要。CopyByte使用了finally块如果error发生的话，保证了两个流关闭，避免资源泄露。
  2. 可能发生的错误：CopyByte不能打开一个或多个文件，当错误发生的时候，流变量对应的文件不会从初始null数值改变。这就是为什么要加一个if判断。

什么时候不用byte stream

copyByte代表了一种你应该避免的低级的I/O。因为xanadu.txt包含字符数据，所以最好使用character stream.

byte stream应该仅仅被用于primitive I/O。其他所有的stream type都基于byte stream。

Character Streams

java平台存储字符使用的是UNICODE。character stream I/O从本地字符集自动翻译这种内在的格式。如果全球化是首要考虑的因素，你应该更加关注于字符集，否则没必要。

Using Character Streams

所有的字符流都继承与Reader，Writer。

copyBite,CopyCharacters都适用int变量读取和写入值。然而，在copyCharacters中，int变量在它的后16位保存字符值，而copybyte中，int变量在它的后8位保存字节值。

使用字节流的而字符流

字符流经常wraper字节流。字符流使用字节流执行物理层I/O，字符流负责字符和字节的转化。比如FileReader使用FileInputStream,FileWriter使用FileOutputStream。

两种常用的字节转化为字符的流，inputStreamReader，OutputStreamWriter。当前有预先打包的字符流类满足需求，可用他们创建字符流。之后用于套接字socket。

基于行的I/O

字符I/O常用于有行终止符的字符串，如回车,”\r\n”,”\r”,”\n”.

buffered stream

上诉例子都使用了没有缓存的I/O，意味着读取或写入都直接与底层系统交互。使得程序不高效，因为每一个请求都出发底层访问，网络活动。

Buffered Input从内存区域buffer，读取数据，当buffer为空的时候，本地输入API调取一次。同样，Buffered output当数据写入buffer，当buffer满的时候执行一次，write操作。

缓存流类包装非缓存流的四种类：BufferedInputStream ,BufferedOutputStream 用于缓存字节流，BufferedReader , BufferedWriter缓存字符流。

Flushing Buffered Streams

在一些关键节点，将数值写进buffer不用等到它满。这就是缓存中的flushing。

一些类的构造器中就有autoflush参数，如PrinterWriter.

可以调用flush方法来手动刷新流。flush方法在输出流是合法的。

Scanning

scanner类型对象：分解格式化的输入为令牌，根据他们的数据类型转化单个的令牌。

Data stream

数据输出流支持初始数据类型(boolean, char, byte, short, int, long, float, and double)和字符串的字节型I/O。所有的数据流都继承了DataInput,DataOutput接口。最常用的的数据流是DataInputStream,DataOutputStream.

DataStream使用的是浮点数代表货币值，这样很不精确。一般java.math.BigDecimal用于货币值的计算，但是bigDecimal用于对象流。

Object Streams

Object Stream支持对象I/O.大多数标准类支持它们对象的序列化。

对象流的类是 ObjectInputStream , ObjectOutputStream,这些类继承了ObjectInput,ObjectOutput接口，它们是DataInput，DataOutput的子接口。所以对象流包括了primitive，object数值。

java.io.RandomAccessFile

因为继承了dataOutput,dataInput接口,所以支持读写.random access file就像一个存储在文件系统中大的字节数组,file-pointer就相当于游标,打开文件,使用seek()定位file-pointer,读写文件.允许非序列化,任意的访问.
