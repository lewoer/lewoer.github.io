Package Static and Final Keywords

发表于 2018-12-31 | 分类于 the complete java developer course | | 热度 ℃

Packages

package： group class in a collection.在组织和分离其他人的库时很方便.使用package的主要原因是为了保持类名的唯一性。

标准的java package是分层次等级的，书写的时候一般为域名的反转，如com.java.Employee

class importation

  • 在类名前添加完整的package name
  • 使用import

在package中定位类的位置是归编译器。

static import

import允许static field or method的导入。

Addition of a Class into a Package

如果不指定package，那么类就属于default package。

添加新的类时可以直接com.mycompany.Scope,Scope是类名

运行特定的类

Caution: 在编译的时候，编译器并不会检查package目录。

package scope

public: 能被所有的类使用
private: 能被定义他们的类使用
package-private: 类，方法，变量能被这个package的所有方法访问。

添加package为library并调用

  • 删除Main方法，打开project structure->library->java
  • 添加需要的package，确定下去。
  • bulid project-> build artifact
  • project structure->artifact

static

static field

如果你定义一个field为static，那么每一个类中，只有一个这样的field。它属于类，而非一个独立的对象。

static constant

static constant比static variable更加常见,它们都为static final

  1. PI
  2. System.out

static method

定义： 不需要操作对象的方法，如Math.pow(x,a)，它们没有隐形参数。

static method 可以访问static field，但是不能访问instance field。

使用static method的两种情况：

  • 方法不需要访问对象的状态，因为所有的参数都是显性参数（Math.pow）
  • 方法仅仅需要访问类中的static field（Employee.getNextId()）

使用static Factory method构建对象，比如说 LocalDate;NumberFormat

每一个类都有main方法，可以添加main方法进行unit test.

final

  • final 常用于private field或是immutable class。
  • final class不能被继承，final method不能被子类重写
  • final field在对象创建之后不能再改变，然而一旦类被声明为final，只有method为final，field不为final。
  • 使用final的好处: 确保在subclass中不会发生语义的改变。

当一个存储密码方法不是final时，当类被继承，重写该方法可以获取密码，造成安全漏洞。

static initializer

static block执行在任何 NON-STATIC前面
